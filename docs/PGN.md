# PGN/SAN

PGN в проекте добавлен как прикладной слой над уже существующим ядром правил.

Главная цель на этом этапе — не полная совместимость со всеми архивами партий, а рабочий импорт/экспорт обычных партий для GUI, ручной отладки и будущих тестовых наборов.

## Что поддерживается

Модуль находится здесь:

```text
src/pgn.rs
```

Он умеет:

- читать теги PGN;
- брать стартовую позицию из тега `FEN`, если партия начинается не из стандартной позиции;
- игнорировать комментарии в `{...}`;
- игнорировать строковые комментарии после `;`;
- игнорировать варианты в скобках `(...)`;
- читать SAN-ходы вроде `e4`, `Nf3`, `Rae1`, `exd5`, `O-O`, `O-O-O`, `Qxf7#`;
- читать UCI-ходы вроде `e2e4` как удобное расширение для внутренних сценариев;
- экспортировать партию обратно в нормальный PGN с SAN;
- сохранять `SetUp` и `FEN` при экспорте партий не из стартовой позиции.

## Что пока не считается целью

Пока не реализована полная PGN-экосистема:

- несколько партий в одном файле;
- сохранение комментариев и вариантов при повторном экспорте;
- NAG как данные, а не как игнорируемые токены;
- часы, оценка, engine annotations;
- метаданные турниров сверх обычных тегов;
- строгий режим валидации всей спецификации PGN.

Это сознательное ограничение. Для текущего движка важнее, чтобы PGN был простым, проверяемым и завязанным на реальные легальные ходы ядра.

## CLI

Проверить PGN-файл можно так:

```bash
cargo run --bin rchess -- pgn game.pgn
```

Или через stdin:

```bash
cat game.pgn | cargo run --bin rchess -- pgn
```

Команда выводит:

- результат партии;
- финальный FEN;
- список ходов в UCI;
- нормализованный PGN.

## GUI

В GUI добавлен блок `PGN`.

`Export PGN` берёт текущую партию из истории ходов GUI и формирует PGN.

`Copy PGN` копирует содержимое PGN-поля в буфер обмена. Если поле пустое, GUI сначала экспортирует текущую партию.

`Load PGN` читает текст из поля PGN, проверяет все ходы через ядро и переводит доску в финальную позицию партии.

`Open PGN file` и `Save PGN file` работают через ручное поле пути. Это не нативный файловый диалог, а простой MVP без новой зависимости.

Если PGN содержит `FEN`, GUI запоминает эту позицию как стартовую для дальнейшего экспорта.

## Принцип

PGN не должен быть отдельной правдой о партии. Правда — это последовательность легальных ходов, применённых к `Position`. Поэтому импорт PGN не просто парсит текст, а каждый ход сопоставляет с легальными ходами текущей позиции.

## SAN-история

Для GUI добавлен публичный помощник `moves_to_san`, а список ходов в правой панели теперь строится через SAN, а не через UCI-строки. UCI-строки остаются внутренним форматом ходов и видны в engine log.

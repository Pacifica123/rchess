# Надёжность ядра

Этот документ фиксирует, что именно сейчас проверяется в ядре правил.

## Базовые проверки

В `src/chess.rs` есть тесты на:

- roundtrip стартового FEN;
- `perft` стартовой позиции до глубины 3;
- `perft divide` и сверку суммы divide с обычным `perft`;
- `perft divide` для нескольких известных позиций, включая сверку суммы depth 2;
- несколько известных `perft`-позиций до глубины 2;
- установку en passant square после двойного хода пешкой;
- применение en passant со снятием битой пешки;
- применение рокировки с переносом ладьи и снятием прав рокировки;
- запрет рокировки через атакованное поле;
- снятие castling rights при ходе ладьи, ходе короля и взятии ладьи на исходной клетке;
- promotion с выбором фигуры;
- генерацию всех четырёх promotion-вариантов на тихом ходе и на взятии;
- pinned piece, который не может уйти и открыть шах своему королю;
- pinned rook, которому разрешены только ходы вдоль линии связки;
- pinned knight, у которого нет легальных ходов из-за связки;
- discovered check после ухода блокирующей фигуры;
- двойной шах, где легальными остаются только ходы королём;
- double check, возникающий после реального хода;
- en passant за чёрных;
- запрет en passant, если он вскрывает шах своему королю;
- мат;
- пат;
- halfmove clock: рост на тихих ходах и сброс на ходе пешкой или взятии.

В `src/pgn.rs` есть тесты на:

- импорт обычной SAN-партии;
- экспорт базового SAN;
- импорт PGN с `FEN` и ходом чёрных первым.

## Почему именно perft

`perft` не измеряет силу движка. Он проверяет генератор ходов. Если значения не совпадают с известными позициями, значит ядро нарушает правила шахмат, и поиск лучшего хода поверх такого ядра становится бессмысленным.

## Следующие проверки

Ближайшие полезные добавления:

- `perft divide` depth 3 для отдельных позиций, когда скорость генератора станет стабильнее;
- тесты на невозможность взятия короля;
- больше позиций с underpromotion;
- больше позиций с en passant, где взятие запрещено из-за вскрытого шаха;
- тесты PGN на неоднозначные SAN-ходы;
- тесты на 50-move rule пока не нужны для генератора ходов, но понадобятся для полноценного результата партии.

## Текущий инкремент

Добавлен `Position::perft_divide(depth)`, CLI-команда `divide`, тест сверки суммы divide с обычным `perft`, а затем расширены edge-case проверки: несколько divide-позиций, pinned rook, pinned knight, discovered check, double check после реального хода, castling rights и halfmove clock.

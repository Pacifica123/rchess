# Experience book

`rchess` теперь поддерживает отдельный режим игры с книгой собственного опыта. Это не нейросеть, не NNUE и не обучение весов оценки. Книга опыта — это обычный текстовый журнал партий и агрегируемая статистика по парам `позиция -> ход`.

Базовый режим без книги не меняется. Если `UseExperienceBook=false`, движок выбирает ход только поиском, как раньше.

## Что считается опытом

После engine-vs-engine матча GUI может дописать партию в файл книги. Каждая строка хода содержит:

```text
move	ply=17	side=w	key=<repetition-key>	fen=<before-fen>	move=e2e4	san=e4	result=win	before_cp=...	after_cp=...	loss_cp=...	accuracy=...	eval_error_cp=...	reason=...
```

Ключ `key` совпадает с шахматным repetition-key: расстановка, сторона хода, рокировки и легальная en-passant цель. Halfmove/fullmove не входят в ключ, потому что книга должна узнавать одну и ту же шахматную ситуацию из разных партий.

Поля `loss_cp`, `accuracy` и `reason` дают простое объяснение: удержал ли ход оценку, потерял ли немного, потерял много или привёл к решающей просадке. Если перед экспортом был запущен нормальный анализ партии, используются его оценки. Если анализа нет, GUI пишет fallback-оценки через текущую тактическую оценку ядра.

## Как книга влияет на выбор хода

Алгоритм намеренно жёстко ограничен:

1. Движок сначала считает root-кандидатов обычным поиском.
2. Лучший ход поиска остаётся главным ориентиром.
3. Книга рассматривает только ходы, которые находятся внутри `ExperienceScoreToleranceCp` от лучшего root-score.
4. Для хода должно быть хотя бы `ExperienceMinGames` партий в книге.
5. Среди допустимых ходов выбирается лучший по обычной статистике результата и средней потере оценки.
6. Все ничьи случаи решаются детерминированно: лучший опыт, больше партий, лучший root-score, затем исходный root-порядок.

Таким образом книга не может протащить заведомо плохой ход только потому, что когда-то он случайно выиграл. Она работает как понятный тай-брейкер между ходами, которые поиск уже считает близкими.

## UCI-настройки

Внутренний UCI `rchess` объявляет такие опции:

```text
UseExperienceBook          check, default false
ExperienceBookPath         string, default rchess_experience.rxp
ExperienceMinGames         spin, default 2
ExperienceScoreToleranceCp spin, default 25
```

Пример:

```text
setoption name UseExperienceBook value true
setoption name ExperienceBookPath value data/rchess_experience.rxp
setoption name ExperienceMinGames value 3
setoption name ExperienceScoreToleranceCp value 30
```

При активной книге движок печатает `info string experience ...`, где указывает выбранный ход, базовый ход поиска, root-score и статистику записи.

## GUI

В правой панели, в блоке `Search resources`, есть секция `Experience book`. Там включается режим книги, задаётся файл или уже существующая папка, минимальное число партий и допустимый разрыв в centipawn. Если путь указывает на папку, движок читает все файлы из неё в стабильном порядке, а GUI при экспорте дописывает в `rchess_experience.rxp` внутри этой папки.

В блоке `Engine vs engine` есть кнопка `Append match to experience book`. Она дописывает текущий матч в файл книги. Для более подробных оценок стоит сначала запустить `Game analysis`, затем отправить матч в книгу.

## Почему это не нарушает философию проекта

Книга опыта не меняет оценочную функцию, не содержит скрытой модели и не добавляет случайность. Это текстовая статистика, которую можно открыть, проверить, удалить, пересобрать или сравнить. Решение остаётся воспроизводимым: одинаковая позиция, одинаковый файл книги и одинаковые настройки дают одинаковый ход.

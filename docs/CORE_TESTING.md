# Надёжность ядра

Этот документ фиксирует, что именно сейчас проверяется в ядре правил.

## Базовые проверки

В `src/chess.rs` есть тесты на:

- roundtrip стартового FEN;
- `perft` стартовой позиции до глубины 3;
- несколько известных `perft`-позиций до глубины 2;
- установку en passant square после двойного хода пешкой;
- применение en passant со снятием битой пешки;
- применение рокировки с переносом ладьи и снятием прав рокировки;
- запрет рокировки через атакованное поле;
- promotion с выбором фигуры;
- мат;
- пат.

В `src/pgn.rs` есть тесты на:

- импорт обычной SAN-партии;
- экспорт базового SAN;
- импорт PGN с `FEN` и ходом чёрных первым.

## Почему именно perft

`perft` не измеряет силу движка. Он проверяет генератор ходов. Если значения не совпадают с известными позициями, значит ядро нарушает правила шахмат, и поиск лучшего хода поверх такого ядра становится бессмысленным.

## Следующие проверки

Ближайшие полезные добавления:

- `perft divide`, чтобы видеть вклад каждого первого хода;
- отдельные тесты на pinned pieces;
- тесты на двойной шах;
- тесты на невозможность взятия короля;
- тесты на сброс halfmove clock;
- больше позиций с promotion и underpromotion;
- больше позиций с en passant, где взятие запрещено из-за вскрытого шаха;
- тесты PGN на неоднозначные SAN-ходы.

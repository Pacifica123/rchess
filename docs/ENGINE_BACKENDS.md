# Engine backends

GUI не должен быть жёстко привязан к одному поиску. Текущая граница остаётся UCI-процессом: интерфейс отправляет позицию, получает `bestmove` и не знает, как именно движок считает ход.

Сейчас в GUI есть три режима:

```text
rchess internal UCI       — наш движок, запущенный как дочерний процесс через --engine-mode
Stockfish 10             — vendored external engine, если его собрать в исполняемый файл
Custom UCI executable    — любой другой UCI-совместимый движок по ручному пути
```

Это не меняет принцип проекта. Собственный движок остаётся главным объектом разработки. Stockfish подключён как внешний ориентир и инструмент сравнения, а не как замена ядра.

## Stockfish 10

В дерево проекта добавлен исходный код загруженной версии Stockfish:

```text
third_party/stockfish-sf_10/
```

Это C++-код и он не собирается через `cargo`. Для GUI нужен готовый исполняемый файл:

```text
third_party/stockfish-sf_10/src/stockfish
third_party/stockfish-sf_10/src/stockfish.exe
```

Если файл лежит в одном из ожидаемых мест, GUI может найти его кнопкой `Detect Stockfish 10`. Кнопка `Use third_party path` просто подставляет стандартный ожидаемый путь, но не собирает движок.

Пример сборки на Unix-like системе из каталога `third_party/stockfish-sf_10/src`:

```bash
make build ARCH=x86-64
```

На Windows это тоже возможно через окружение, где есть `make` и C++ compiler, например MSYS2/MinGW. В противном случае можно собрать Stockfish отдельно и указать путь к `stockfish.exe` вручную.

## Зачем это нужно

Практическая цель — получить внешний сильный UCI-движок рядом с нашим GUI. Это даёт:

- проверку, что GUI умеет работать не только с нашим UCI;
- быстрый контроль позиции сильным движком;
- основу для будущего режима `engine vs engine`;
- возможность сравнивать наш поиск со стабильным внешним ориентиром.

Будущий режим противостояния движков лучше строить как два независимых UCI-слота:

```text
White engine process  <->  arbiter/game controller  <->  Black engine process
```

Контроллер партии должен владеть доской, часами, PGN и правилами завершения. Движки должны только получать позицию и возвращать ход.

## Лицензия

Stockfish распространяется под GNU GPL. Оригинальный файл лицензии лежит здесь:

```text
third_party/stockfish-sf_10/Copying.txt
```

Если проект распространяется вместе с этим каталогом, нужно учитывать GPL-условия Stockfish. Собственное Rust-ядро не должно импортировать C++-код Stockfish и не должно зависеть от него при сборке.
